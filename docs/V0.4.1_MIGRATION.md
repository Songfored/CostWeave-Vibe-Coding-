# CostWeave v0.3 → v0.4.1 迁移说明

v0.4.1 不需要数据库或模型目录迁移。

## 升级

1. 使用 v0.4.1 源码替换程序文件。
2. 保留原模型目录文件；其 Schema 没有变化。
3. 运行 `python -B -m unittest discover -s tests -v`。
4. 启动 `python -m costweave.server`。
5. 检查 `/api/health`：
   - `version` 为 `0.4.1`
   - `contract_schema` 为 `v4.1`
   - `features.contracts_v4` 为 `true`
   - `contract_trace_persistent` 为 `false`

新创建的运行 ID 会从 12 位旧格式变为 `run_<32 hex>`。原 API 路径仍接受返回的完整 ID。

## 临时回滚

无需改动数据。设置环境变量后重启服务：

```powershell
$env:COSTWEAVE_CONTRACTS_V4="0"
python -m costweave.server
```

此时：

- 引擎恢复 v0.3 直接状态赋值语义
- 新运行恢复 12 位旧 ID
- 运行快照不附加 `contract` 字段
- 不产生 v4 状态踪迹

删除环境变量或设为 `1` 后即可重新启用。

## 数据边界

- v0.4.1 没有运行数据库。
- 进程内合同踪迹不会写入磁盘。
- 模型目录数据格式没有变化。
- v0.4.1 产生的 API 响应无需降级迁移；回滚开关会恢复旧响应形状。
