# CostWeave

**本地优先、成本感知的多模型 Agent 编排运行时雏形。**

CostWeave 在执行任务之前判断难度和能力需求，把复杂目标转换成结构化任务合同与并行 DAG，预测式选择执行者，并在关键交接处验收结果。如果半程证据表明核心假设错误，它会保留有效成果、替换错误节点并重新规划。

> v0.1 不调用任何真实 AI 模型或外部 API。它使用模拟执行器展示完整产品控制面，因此克隆后可以零配置运行。真实模型接入被刻意留作下一阶段。

## 功能

- 本地启发式总指挥：任务类型、L1—L5难度和风险粗判
- 预测式路由：执行前比较能力匹配、成功率、成本和延迟
- 结构化任务合同：目标、范围、依赖、Schema和验收标准
- 异步 DAG：无依赖节点并行，完成即动态释放下游
- 逐节点验收：只有通过验证的成果能够进入关键下游
- 动态成功率：根据真实执行证据持续更新
- 三级恢复模型：数据结构支持节点、子图和全局恢复
- 全局重规划演示：推翻错误计划但复用已验证成果
- 预算、最大并发、质量门槛和运行模式控制
- 运行时间线、任务图、任务合同和成本指标界面
- 零第三方运行依赖

## 快速开始

需要 Python 3.11 或更高版本。

```bash
python -m costweave.server
```

打开 `http://127.0.0.1:8765`。

也可以安装为本地命令：

```bash
python -m pip install -e .
costweave
```

## 测试

```bash
python -m unittest discover -s tests -v
```

测试覆盖任务识别、DAG合法性、模型预路由、异步并行、预算守卫、半程失败重规划和HTTP API。

## 演示建议

在界面中输入包含多个领域的任务，例如：

```text
开发一个市场数据分析应用，完成资料研究、技术架构、风险分析并形成最终报告。
```

第一次可以直接运行，观察研究、开发、数据和写作分支并行执行。第二次勾选“演示中途重规划”，系统会注入一次核心假设错误，展示：

1. 节点验收拒绝结果；
2. 成功概率下降；
3. 已验证成果保留；
4. 高级规划顾问替换失败节点；
5. 依赖自动重连；
6. 新计划恢复执行并完成。

## 项目结构

```text
costweave/
├── domain.py       # 运行、任务合同和状态模型
├── planner.py      # 本地启发式规划器与重规划
├── router.py       # 成本/质量/延迟感知的预测式路由
├── validator.py    # DAG、合同和结果验收
├── executor.py     # 可替换的离线模拟执行器
├── engine.py       # 异步DAG调度、预算与恢复
├── server.py       # 零依赖HTTP API与静态服务
└── web/            # 产品界面
```

详细设计见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)。

## API

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/api/health` | 运行状态 |
| `GET` | `/api/catalog` | 模拟执行者能力目录 |
| `POST` | `/api/runs` | 创建编排任务 |
| `GET` | `/api/runs/{id}` | 获取计划、节点和事件 |
| `GET` | `/api/runs` | 最近运行记录 |

创建任务示例：

```json
{
  "goal": "开发一个市场分析产品并形成报告",
  "mode": "balanced",
  "budget": 1.0,
  "quality_floor": 0.78,
  "max_concurrency": 4,
  "simulate_replan": true
}
```

## 当前边界

- 启发式规划不是训练后的本地模型。
- 执行者只返回模拟结构化成果，不回答真实领域问题。
- 成功概率是规则演示值，尚未通过历史任务数据校准。
- 状态保存在内存中，服务重启后不会保留。
- 全局重规划在MVP中最多触发一次。

这些边界都被放在明确的模块接口之后，不妨碍当前版本用于展示、测试和后续扩展。

## 路线图

- 本地规划模型与置信度校准

- OpenAI-compatible多厂商适配器

- 基于历史数据的成功率预测
- 节点级、子图级和人工审批恢复策略
- 真实基准任务上的质量—成本—延迟评测
