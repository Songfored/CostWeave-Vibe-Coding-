# CostWeave v0.4.1

发布日期：2026-07-19

## 新增

- 统一、可验证的 `RunId`、`TaskId`、`EventId` 和 `ArtifactId`
- 兼容旧语义 ID 的确定性 v4 映射
- 版本化 `CommandEnvelope` 和 `EventEnvelope`
- 严格 JSON、Schema、ID、事件类型和时间戳校验
- 显式运行与任务状态转换表
- 纯函数状态转换守卫、终态保护和幂等同状态写入
- 带来源与拒绝原因的进程内只读状态踪迹
- v0.3 运行快照到 v4 合同视图的无损兼容外壳
- `COSTWEAVE_CONTRACTS_V4` 回滚开关
- `/api/health` 合同 Schema、开关状态和非持久化边界

## 调整

- 项目版本升级为 `0.4.1`
- 新运行默认使用 `run_<32 hex>` ID
- 引擎中的运行和任务状态变更统一经过兼容守卫
- README、架构说明和增量发布路线更新为 v0.4.1 基线

## 兼容性

- 原 v0.3 API 路径和主要响应字段保持不变
- 模型目录 Schema 和持久化文件不变
- 设置 `COSTWEAVE_CONTRACTS_V4=0` 可恢复 v0.3 状态赋值与快照形状
- 无数据库迁移、无新增后台服务、无真实模型调用

## 验证

- 自动化测试：60 / 60 通过
- 新增合同与兼容测试：19 项
- 功能开关开启和关闭两种完整运行路径均通过
- HTTP、模型目录、规划、路由、并行、验收、重规划和页面回归全部通过
- PEP 517 wheel 构建成功，并在独立目标目录验证版本、合同模块和网页资源

## 已知边界

- 状态踪迹仅存在于当前进程，重启后丢失
- v0.4.1 不提供 EventStore、快照重放、幂等命令持久化或启动恢复
- 这些能力属于 v0.4.2
